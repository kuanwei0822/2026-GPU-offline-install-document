# 工作流程與角色定義

## 角色定義

- 👤 人為操作
- 🤖 Claude Code
- 🔄 Loop 直到條件達成

---

## 核心設計原則

**步驟1（目標機收集）與步驟6（目標機安裝）進入客戶環境的次數要盡量少。**

因此：

- **步驟2｜驗證 sysinfo.json**
  - 一次性找出所有問題，不能只發現一個問題就要求重跑
  - 驗證完必須給出完整的問題清單，一次修正 `collect.sh` 到位
  - 目標：步驟1 最多跑兩次（第一次收集、最多一次補跑）

- **步驟7｜失敗處理**
  - 一次性分析所有錯誤，不能只處理一個問題就要求重試
  - 所有情況（套件、腳本、資訊不足）同時處理，給出完整修正方案
  - 資訊不足時，一次提出所有需要查詢的問題，不能來回多輪才問完
  - 目標：步驟6 重試次數盡量控制在一次以內

---

## 完整工作流程

### 步驟1｜目標機收集資訊
- 👤 在目標機執行 `collect.sh`
- 👤 將產生的 `sysinfo.json` 帶出來

### 步驟2｜驗證 sysinfo.json
- 🤖 自動驗證 `sysinfo.json` 完整性
- ✅ 驗證通過 → 前往步驟3
- ❌ 驗證失敗
  - 🤖 自動修正 `collect.sh`，產生新版本
  - 👤 將新版 `collect.sh` 帶回目標機
  - 🔄 回步驟1，直到驗證通過

### 步驟3｜下載套件
- 🤖 根據 `sysinfo.json` 分析需要的套件
- 🤖 下載所有套件（寧可多帶，不要少帶）

### 步驟4｜靜態驗證
- 🤖 驗證套件完整性
- 🤖 驗證版本吻合
- 🤖 rpm --test 模擬安裝
- 🤖 驗證編譯環境齊全
- 🤖 **CHECK 8（必要）**：傳入 `sysinfo.json`，驗證 GPU 架構與 kernel module 類型相符
  - Blackwell / Hopper → 必須有 `kmod-nvidia-open-dkms`，否則停止並要求重新下載
- ✅ 全部通過 → 前往步驟5
- ❌ 有問題
  - 🤖 補充下載缺少的套件
  - 🔄 重試步驟4，直到全部通過

### 步驟5｜產生安裝腳本
- 🤖 根據 `sysinfo.json` 與套件清單產生 `install.sh`
- 🤖 `install.sh` 支援斷點續裝

### 步驟6｜目標機安裝

**6-1. 執行安裝腳本**
- 👤 將套件與 `install.sh` 帶入目標機
- 👤 執行 `install.sh`
- ✅ 腳本跑完 → 前往 6-2
- ❌ 第一次失敗 → 帶出 `install_XXX.log`，前往步驟7（情況A/B/C）
- ❌ **第二次（含）以上失敗 → 直接進入步驟7 情況D**，不再產生新版腳本

**6-2. 重開機**
- 👤 `sudo reboot`
- 👤 等待開機完成

**6-3. 重開機後驗證**
- 👤 執行驗證指令：
  ```bash
  nvidia-smi
  dkms status
  docker run --rm --gpus all nvidia/cuda:XX.X-base nvidia-smi
  ```
- ✅ 全部通過 → 安裝完成，結束
- ❌ 驗證失敗 → 前往步驟7 **情況D**（重開機後失敗通常需要即時排查）

### 步驟7｜失敗處理
- 🤖 分析 `install.log`（含各步驟驗證結果、state snapshot、error 內容）

**情況A｜套件問題**
- 🤖 給出補充下載建議
- 👤 確認建議
- 🔄 回步驟3

**情況B｜腳本問題**
- 🤖 修正 `install.sh`
- 👤 確認修正
- 🔄 重試步驟6

**情況C｜資訊不足**
- 🤖 提出需要查詢的資訊清單
- 👤 在目標機或相關環境查詢並回覆
- 🔄 雙方來回，直到資訊足夠
- 👤 修正 `collect.sh`（獨立進行，下次帶入客戶環境）
- 🤖 修正 `install.sh`
- 👤 判斷回到哪個步驟
  - 需要重新下載套件 → 回步驟3
  - 套件沒問題 → 重試步驟6

**情況D｜需要手動排查**
- 適用場景：問題複雜、原因不明，需要即時在目標機上蒐集資訊才能判斷
- 🤖 請執行人執行指定指令（診斷、查詢系統狀態、安裝單一套件等）
- 👤 將指令輸出或說明以文字回覆 Claude
- 🔄 Claude 根據回覆分析，再提出下一步指令或方向
- 🔄 雙方以對話方式持續推進，直到問題解決或確認無法解決
- 執行人可用自然語言描述現象，Claude 依描述調整診斷方向
- ✅ 問題解決 → 判斷回到哪個步驟，或直接繼續
- ❌ 確認無法解決 → 記錄為已知限制，評估替代方案
